# Aula 02 - O que é programação

## Lógica de Programação do Zero ao Avançado

---

# 1. Apresentação da Aula

Antes de aprender qualquer linguagem de programação, precisamos responder uma pergunta fundamental:

**O que é programação?**

Muitas pessoas imaginam que programar é apenas escrever códigos em uma tela.

Essa é uma visão incompleta.

Programar é muito mais do que escrever comandos.

Programar é criar uma sequência de instruções que permite que um computador execute uma tarefa e resolva um problema.

O código é apenas a forma como comunicamos essa solução para o computador.

Antes do código existe o pensamento.

---

# 2. Explicação Teórica

Programação é o processo de criar instruções que orientam um computador a realizar determinada atividade.

Um programa é formado por uma sequência de passos organizados.

Esses passos informam:

- O que deve ser recebido.
- O que deve ser feito.
- Qual decisão deve ser tomada.
- Qual resultado deve ser apresentado.

Um programador transforma uma necessidade em uma solução que pode ser executada por uma máquina.

---

# 3. Exemplo do Cotidiano

Imagine uma pessoa explicando para outra como preparar um café.

Ela não diria apenas:

"Faça café."

Essa informação é muito vaga.

Ela precisaria explicar:

1. Pegar uma xícara.
2. Colocar água para aquecer.
3. Preparar o café.
4. Adicionar açúcar se desejar.
5. Servir.

Perceba:

Existe uma sequência de passos.

Existe uma ordem.

Existe uma finalidade.

Isso é uma forma de algoritmo.

Na programação acontece algo parecido.

A diferença é que o computador precisa de instruções muito mais precisas.

---

# 4. Explicação Técnica

Um programa de computador normalmente possui três partes fundamentais:

## Entrada

São os dados que o programa recebe.

Exemplos:

- Nome do usuário.
- Idade.
- Valor de uma compra.
- Notas de um aluno.

---

## Processamento

São as operações realizadas pelo programa.

Exemplos:

- Fazer cálculos.
- Comparar informações.
- Organizar dados.
- Tomar decisões.

---

## Saída

É o resultado apresentado.

Exemplos:

- Uma mensagem na tela.
- Um relatório.
- Um valor calculado.

---

Representação:

ENTRADA

Dados recebidos
   ↓

PROCESSAMENTO

   ↓

SAÍDA

Resultado apresentado


Esse conceito será utilizado durante todo o curso.

---

# 5. Exemplo Prático

Problema:

Criar um programa que calcule a média de um aluno.

Antes do código, pensamos:

## Entrada

Precisamos receber:

- Nota 1.
- Nota 2.

---

## Processamento

Precisamos:

- Somar as notas.
- Dividir por 2.

---

## Saída

Mostrar:

- A média do aluno.

---

Observe:

O programador primeiro entende o problema.

Depois cria a solução.

Somente depois transforma em código.

---

# 6. Como um programador pensa

Um iniciante normalmente pensa:

"Qual código eu preciso escrever?"

Um programador experiente pensa:

"Qual problema preciso resolver?"

Essa diferença é fundamental.

O código é uma consequência do raciocínio.

---

# 7. Erro comum de iniciantes

Um erro muito comum é tentar aprender programação decorando comandos.

Exemplo:

"Qual comando mostra uma mensagem?"

"Qual comando faz uma repetição?"

"Qual comando cria uma variável?"

Esses conhecimentos são importantes, mas vêm depois.

Primeiro precisamos aprender:

Como pensar.

Como analisar.

Como resolver.

---

# 8. Exercício Guiado

Vamos analisar um problema simples.

Problema:

"Um sistema precisa informar a idade de uma pessoa."

Responda:

## Qual é a entrada?

Resposta esperada:

Uma idade.

---

## Qual é o processamento?

Resposta esperada:

Receber e armazenar a idade.

---

## Qual é a saída?

Resposta esperada:

Mostrar a idade informada.

---

# 9. Exercício Individual

Analise o seguinte problema:

"Um aplicativo de banco permite consultar o saldo de uma conta."

Responda:

1. Qual é a entrada?

2. Qual é o processamento?

3. Qual é a saída?

---

# 10. Desafio

Explique com suas palavras:

Por que programar não é apenas escrever código?

---

# 11. Resumo da Aula

Nesta aula aprendemos:

- O conceito de programação.
- Que programação é resolver problemas.
- Que código é apenas uma forma de comunicar uma solução.
- O conceito de entrada, processamento e saída.
- Como um programador deve pensar antes de escrever código.

---

# 12. Revisão

Antes de criar um programa, sempre pergunte:

Qual problema estou resolvendo?

Quais informações entram?

O que precisa acontecer?

Qual resultado deve sair?

Programadores não começam pelo código.

Programadores começam pelo problema.
