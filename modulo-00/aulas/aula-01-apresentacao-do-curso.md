# Aula 01 - Apresentação do Curso

## Lógica de Programação do Zero ao Avançado

---

# 1. Apresentação da Aula

Bem-vindo ao curso:

**Lógica de Programação do Zero ao Avançado**

Este curso foi criado com um objetivo principal:

Ensinar você a pensar como um programador.

Durante esta jornada, você não aprenderá apenas comandos de uma linguagem de programação.

Você aprenderá como analisar problemas, organizar pensamentos e criar soluções utilizando lógica.

A programação começa antes do código.

Antes de escrever qualquer programa, um desenvolvedor precisa saber:

- Qual problema precisa resolver.
- Quais informações são necessárias.
- Quais passos devem ser executados.
- Qual resultado deve ser produzido.

Este curso foi estruturado para construir essa base.

---

# 2. Objetivo do Curso

O objetivo deste curso é desenvolver no aluno a capacidade de:

- Analisar problemas.
- Criar algoritmos.
- Organizar soluções.
- Desenvolver raciocínio lógico.
- Resolver problemas computacionais.
- Aprender qualquer linguagem de programação com maior facilidade.

O foco principal não será decorar código.

O foco será desenvolver a habilidade de pensar como um programador.

---

# 3. Por que aprender lógica antes de programar?

Muitas pessoas começam programação tentando decorar comandos:

"Qual comando faz isso?"

"Qual código resolve esse problema?"

Esse caminho normalmente gera dificuldade.

Um programador profissional primeiro entende o problema.

Depois cria uma solução.

Somente então transforma essa solução em código.

A lógica é a ponte entre o problema e o programa.

Exemplo:

Problema:

"Calcular a média de um aluno."

Antes do código, precisamos pensar:

Entrada:

- Nota 1.
- Nota 2.

Processamento:

- Somar as notas.
- Dividir por 2.

Saída:

- Mostrar a média.

Esse pensamento é a base da programação.

---

# 4. Como será o aprendizado

Este curso seguirá uma metodologia progressiva.

Não iremos pular etapas.

A evolução será:

## Nível 1 — Conceitos básicos

Aprender os fundamentos:

- Entrada.
- Saída.
- Variáveis.
- Operações.
- Decisões.

---

## Nível 2 — Combinação de conceitos

Começar a resolver problemas utilizando vários conceitos juntos.

Exemplo:

Receber dados.

Realizar cálculos.

Tomar decisões.

Mostrar resultados.

---

## Nível 3 — Resolução de problemas

Aprender a analisar problemas maiores.

Dividir problemas complexos em partes menores.

Criar soluções organizadas.

---

## Nível 4 — Problemas próximos do mercado profissional

Aplicar o conhecimento em situações reais.

Exemplos:

- Sistemas.
- Cadastros.
- Processamentos.
- Regras de negócio.

---

# 5. Metodologia do Curso

Durante todo o curso seguiremos alguns princípios:

## Não decorar código

O objetivo não é memorizar comandos.

O objetivo é entender como resolver problemas.

---

## Não copiar soluções sem entender

Um programador precisa saber criar soluções próprias.

Copiar código sem compreender não desenvolve raciocínio.

---

## Não pular etapas

Cada conceito será construído sobre o anterior.

Uma base forte facilita todo o aprendizado futuro.

---

## Aprender através da prática

Programação se aprende praticando.

Por isso teremos muitos exercícios progressivos.

---

## Desenvolver autonomia

O objetivo final é que você consiga resolver problemas sozinho.

---

# 6. Como será cada aula

Cada aula seguirá uma sequência:

1. Explicação do conceito.

2. Exemplo prático.

3. Demonstração passo a passo.

4. Teste de mesa.

5. Exercício guiado.

6. Exercício individual.

7. Correção comentada.

8. Desafio.

Esse processo será repetido durante todo o curso.

---

# 7. A regra principal do curso

Antes de criar qualquer algoritmo, sempre responderemos cinco perguntas:

## 1. Qual é o problema?

Entender exatamente o que precisa ser resolvido.

## 2. Quais são as entradas?

Identificar quais informações serão recebidas.

## 3. O que será processado?

Definir quais cálculos ou transformações serão realizados.

## 4. Existe alguma decisão?

Verificar se existe alguma condição.

## 5. Qual será a saída?

Definir qual resultado será apresentado.

Essas cinco perguntas serão utilizadas durante todo o curso.
